# Orderly UI 样式建议落地计划（v2）

> 计划日期：2026-02-19  
> 输入文档：`docs/UI样式建议.md`  
> 目标：将 v2 样式建议转化为可执行改造，优先提升信息层次、可读性与操作清晰度。

## 1. 实施范围

| 建议编号 | 主题 | 优先级 | 说明 |
|---|---|---|---|
| 1 | 侧边栏比例与品牌感 | P1 | 调整宽度、品牌标题强化 |
| 2 | 搜索栏与筛选区层次 | P0 | 搜索独占一行，筛选分层 |
| 3 | 空状态差异化 | P0 | 左右空状态分工明确 |
| 4 | 右侧详情面板分割与标题栏 | P0 | 固定 toolbar + 分隔增强 |
| 5 | FAB 功能可感知性 | P1 | 蓝色主题 + tooltip + hover 动效 |
| 6 | 列表项 hover/选中态强化 | P1 | 状态区分和过渡动画 |
| 7 | 筛选标签 chip 化 | P1 | 提升筛选可读性与反馈 |
| 8 | 微交互动画 | P2 | 骨架屏、刷新动画、切换过渡 |
| 9 | 暗色模式预埋 | 不纳入本期 | 明确排除（按当前需求不做） |

## 2. 分阶段执行

### Phase A（P0，1-2 天）：信息层次重构

#### A1. 搜索/筛选区重排（建议 2）
- 任务：
  - 搜索框改为独占一行，左侧搜索图标，右侧放刷新 icon 按钮。
  - 筛选区下沉为第二行，统一尺寸/字号（`text-xs`）和浅蓝背景。
  - 日期区从双输入框迁移为日期范围选择组件（先实现简化版，再替换为完整组件）。
- 影响文件（预估）：
  - `src/App.tsx`
  - `src/components/ui/input.tsx`
  - `src/components/ui/select.tsx`
- 验收：
  - 搜索区与筛选区视觉分层明显，移动端/窄窗不溢出。
  - 刷新按钮不抢主视觉且可单独触发。

#### A2. 详情面板标题栏 + 空状态差异化（建议 3、4）
- 任务：
  - 右侧详情区域增加固定标题栏（标题、当前单子名、主操作按钮）。
  - 标题栏下加 `border-b`，面板使用 `bg-card` 或轻阴影增强分区。
  - 左侧空状态改为插图 + 渐变背景；右侧改为“从左侧选择单子”引导，不重复主按钮。
- 影响文件（预估）：
  - `src/App.tsx`
  - `src/styles.css`
- 验收：
  - 双空状态时依然能清楚分辨左右职责。
  - 用户在 3 秒内能识别“下一步该做什么”。

### Phase B（P1，1-2 天）：品牌与交互清晰度提升

#### B1. 侧边栏品牌强化 + FAB 明确化（建议 1、5）
- 任务：
  - 侧边栏宽度调整为 `w-36`（保留可回滚配置）。
  - 品牌标题升级为 `text-lg font-bold`，增加蓝色标识线。
  - FAB 调整为主题蓝，增加 tooltip 与 `hover:scale-110`。
- 影响文件（预估）：
  - `src/App.tsx`
  - `src/styles.css`
- 验收：
  - 首屏品牌识别增强，FAB 行为可理解。
  - 不影响现有布局（列表/详情最小宽度保持可用）。

#### B2. 列表态与筛选 Chip（建议 6、7）
- 任务：
  - 列表项选中态加入左侧主色线，hover 增加背景过渡。
  - 筛选条件改为 chip/pill 展示，激活态增加状态标识（点或徽记）。
- 影响文件（预估）：
  - `src/App.tsx`
  - `src/components/ui/badge.tsx`
  - `src/styles.css`
- 验收：
  - 用户可快速分辨“当前选中项”和“鼠标悬停项”。
  - 激活筛选条件一眼可见。

### Phase C（P2，1-2 天）：体验精修

#### C1. 微交互与骨架屏（建议 8）
- 任务：
  - 列表加载态接入 skeleton，避免白屏感。
  - 刷新时图标旋转动画。
  - 面板切换加入淡入/滑入过渡。
  - 新建后新项短暂高亮。
- 影响文件（预估）：
  - `src/App.tsx`
  - `src/styles.css`
- 验收：
  - 交互反馈连贯，不引入明显性能损耗。

## 3. 里程碑与交付物

| 里程碑 | 时间 | 交付物 |
|---|---|---|
| M1 | T+2 天 | 搜索筛选分层、详情标题栏、空状态差异化 |
| M2 | T+4 天 | 侧边栏品牌强化、FAB 优化、列表态与筛选 Chip |
| M3 | T+6 天 | 微交互动画 |

## 4. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 布局改动导致小窗口挤压 | 可用性下降 | 每阶段做 1280/1024/900 宽度回归 |
| 动画过多影响性能 | 交互卡顿 | 动画仅保留关键路径，统一 `duration <= 200ms` |

## 5. 完成定义（DoD）

- [ ] `docs/UI样式建议.md` 的 P0 项全部落地并可演示
- [ ] 关键页面（主列表/详情/设置）在 1280px 与 900px 以上无布局破坏
- [ ] 交互回归通过（搜索、筛选、刷新、新建、详情查看）
- [ ] 样式改动均集中在 token/组件层，无大量重复 class 粘贴
