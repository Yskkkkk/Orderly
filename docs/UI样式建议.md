## 🔧 仍需改进的问题

### 1. 侧边栏比例与间距

**问题：** 侧边栏固定 `w-40`（160px），对于只有两个菜单项来说偏宽；品牌名 "Orderly" 字号偏小，缺乏品牌感。

**建议：**
- 侧边栏宽度缩减至 `w-36`（144px）或改为可折叠设计
- "Orderly" 标题加大至 `text-lg`，增加左侧蓝色标识线或小 Logo
- 最上方增加 `pt-3` 呼吸空间

```diff
-<div className="text-base font-semibold text-primary">Orderly</div>
+<div className="flex items-center gap-1.5 text-lg font-bold text-primary">
+  <span className="inline-block h-5 w-1 rounded-full bg-primary" />
+  Orderly
+</div>
```

---

### 2. 搜索栏与筛选区层次感不足

**问题：** 搜索框、三个下拉、两个日期框视觉权重相同，排列密集、层次扁平。刷新按钮 `🔄` 虽已改为图标，但视觉上仍与搜索框竞争。

**建议：**
- **搜索框独占一行**，全宽显示，内缩图标式设计（左侧放 🔍 图标）
- 搜索框下方用细 `border-b` 或 `gap` 分隔后展示筛选行
- 三个下拉框统一缩小字号（`text-xs`），使用统一的浅蓝底色（`bg-accent/50`）
- 刷新按钮移至搜索框右侧，改用更小的 `variant="ghost"` + `size="icon"` 样式
- 日期"从→到"改为单组件日期范围选择器（推荐 `react-day-picker` 的 DateRange 模式）

```
┌──────────────────────────────────────────┐
│ 🔍  搜索微信号 / 用户名 / 系统名称   🔄 │  ← 搜索独占一行
├──────────────────────────────────────────┤
│ 状态[全部▾]  视图[正常▾]  时间[最近更新▾] │  ← 筛选紧凑排列
│ 📅 2026-02-12 → 2026-02-19  [清空]      │
└──────────────────────────────────────────┘
```

---

### 3. 空状态视觉吸引力

**问题：** 左侧列表和右侧详情面板各有一个空状态，但两者几乎相同，缺乏差异化。整体虚线边框视觉偏弱。

**建议：**
- 左侧列表空状态：保留但换用更大的插图（SVG illustration），加柔和的渐变背景色
- 右侧详情空状态：改为箭头引导式设计（「← 从左侧选择一个单子」），不重复放"新建单子"按钮（避免与右上角按钮冲突）
- 虚线边框改为 `border-dashed border-primary/20`（浅蓝虚线），对比现有灰色更有品牌感

```diff
 /* 左侧空状态 */
-<div className="border border-dashed rounded-xl ...">
+<div className="border border-dashed border-primary/20 rounded-xl bg-gradient-to-b from-accent/30 to-transparent ...">

 /* 右侧空状态 */
-还没有选中单子
+← 从左侧列表中选择一个单子查看详情
```

---

### 4. 右侧详情面板分割效果

**问题：** 左右面板的分隔仅靠 `border-border/70`，视觉区分度不够。当两侧都为空状态时，页面整体显得空洞。

**建议：**
- 右侧面板增加微弱的 `shadow-sm` 或改变背景色为 `bg-card` 以区分
- "详情" 标题行改为固定 toolbar 样式：左侧放"详情"标题 + 选中单子名，右侧放"新建单子"按钮
- 标题行下方增加 `border-b` 分隔线

```
┌─────────────────────────────────────────┐
│ 📋 详情 — [选中的系统名]       [新建单子] │  ← 固定标题栏
├─────────────────────────────────────────┤
│                                         │
│        ← 选择一个单子查看详情            │
│                                         │
└─────────────────────────────────────────┘
```

---

### 5. 悬浮按钮（FAB）功能不明

**问题：** 右下角的黑色圆形悬浮按钮缺乏说明，用户不知道点击后的行为。

**建议：**
- 添加 hover tooltip（如 `title="快捷操作"`）
- 按钮颜色从黑色改为主题蓝色 `bg-primary`
- 添加 hover 缩放动画（`hover:scale-110 transition-transform`）
- 考虑改为 Speed Dial 模式：点击展开"新建单子"/"刷新列表"等操作

## 🆕 新增建议

### 6. 卡片/行列表悬停效果

**问题：** 当有数据时，列表项的 hover 和选中态区分度可能不够清晰。

**建议：**
- 选中行使用 `bg-accent border-l-2 border-primary` 左侧蓝线
- Hover 行加 `bg-muted/60` 过渡
- 添加微妙的 `transition-all duration-150` 使状态切换更丝滑

---

### 7. 筛选标签增加 chip 样式

**当前：** 下拉标签（"状态"、"视图"、"时间字段"）使用普通文字。

**建议：**
- 改为 chip/pill 样式，选中时有品牌色背景
- 激活的筛选条件加一个小圆点或数字角标

```
  [状态: 全部]   [视图: 正常]   [时间: 最近更新 ▾]
        ↓ 选中某个筛选后
  [状态: 待发送 ●]  [视图: 正常]   [时间: 最近更新 ▾]
```

---

### 8. 微交互动画

**建议添加以下微动画提升整体精致感：**
- 列表加载时：骨架屏（skeleton）替代空白
- 点击刷新时：刷新图标旋转 360°
- 面板切换时：`animate-in fade-in slide-in-from-left` 过渡
- 新建单子后：新行闪烁高亮（flash highlight）

---

### 9. 暗色模式（优先级未变，但可先预埋）

**当前：** `tailwind.config.ts` 已配置 `darkMode: ["class"]`，但 `styles.css` 中无 `.dark` 变量。

**建议先在 `styles.css` 中预埋暗色变量：**

```css
.dark {
  --background: 222 14% 10%;
  --foreground: 210 20% 92%;
  --card: 222 14% 14%;
  --card-foreground: 210 20% 92%;
  --primary: 212 84% 60%;
  --primary-foreground: 0 0% 100%;
  --muted: 222 12% 18%;
  --muted-foreground: 220 10% 55%;
  --accent: 212 60% 20%;
  --accent-foreground: 212 84% 80%;
  --border: 222 10% 22%;
  --input: 222 10% 22%;
  --ring: 212 84% 60%;
}
```

在设置页面添加一个主题切换 Toggle 即可启用。
